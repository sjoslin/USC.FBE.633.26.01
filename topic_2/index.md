# Topic 2 - First Agent Session using API Keys

## Overview

This topic we will have our first agent session to access data. Using manual approvals, we will pull real personal consumption data from FRED and produce a plot illustrating the Great Moderation. We will see the power of agents but also the cost of manual approval required to prevent potentially destructive commands from the agent.

---

## Installing VS Code Extensions

We will use the Claude Code extension for VS Code. Follow these steps:

1. Open VS Code
2. Go to the Extensions view (Ctrl+Shift+X or Cmd+Shift+X)
3. Search for "Claude Code"
4. Click Install
5. Reload VS Code when prompted

Once installed, you can authenticate with your API key and begin using the agent.

## Using API Keys

We will be using data from the St. Louis Federal Reserve's FRED (Federal Reserve Economic Data) dataabase. One way to get this data is through their web interface. Often, a better method is to access the data programmatically through an API (Application Program Interface). This is how we will access the data.

### FRED API Key

In order to give Python (or other programming language) access to our API key, we will use a .env file. On linux-based systems, files beginning with . are hidden by default.

1. In VS Code, create a folder for Topic 2 and open it.
2. In the folder, create a file .env with: FRED_API_KEY=... and save it.
3. Note: it can be useful to open and close the file explorer by clicking the file icon.

## Example: The Great Moderation

We will work through an example using FRED data to illustrate the Great Moderation:

1. Connect to FRED using your API key
2. Pull real personal consumption data and industrial production
3. Generate a plot showing the Great Moderation
4. Review the code generated by the agent
5. Approve or modify commands as needed

This example demonstrates both the power of agents and the importance of manual approval workflows.

Start with a prompt along the following lines (feel free to substitute R for Python)

```text
Use the FRED API key in the .env dir, please download monthly data for personal consumption expenditure for from 1970 to 2024.  Save the data as a csv.  Generate a time series yy-plot of the (overlapping) year on year changes.
```

Claude will do all of this easily! Note however, that is will just randomly write files wherever. This is not great, and we will come back to this.

Now, actually, we want 1950. So ask Claude:

```text
Actually, can you change the start year to 1950.
```

Notice that when it wants to change the file, it shows you the **diff** in the file and asks for your approval.

```
Now we want to examine the Great Moderation. Compute the rolling 5-year volatility of monthly changes.  Let's also use the real PCE instead of nominal.
```

This will almost show the great moderation. There will be a few issues that can be easily addressed.
